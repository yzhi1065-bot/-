# 中医智能诊断系统 — PRD 产品需求文档

版本：V1.0  
日期：2026年6月23日  
状态：初稿

---

## 1. 项目背景

### 1.1 行业背景
中医药学是中华民族的瑰宝，但在现代化进程中面临诊断标准化不足、数据难以量化、经验传承困难等挑战。随着人工智能技术的成熟，将 AI 与传统中医四诊（望、闻、问、切）结合，实现诊断辅助和智能开方，已成为中医现代化的重要方向。

### 1.2 项目目标
构建一个"**采集-分析-诊断-治疗**"一体化的中医智能辅助诊断系统：
- **采集端**：连接硬件设备，采集四诊客观数据 + 支持医生人工录入
- **模型中台**：调用大模型进行证型分析、诊断建议、方剂推荐
- **交付端**：生成结构化诊断报告和治疗方案

### 1.3 核心价值
1. 将中医四诊数字化、标准化、可量化
2. 辅助基层中医师提升诊断准确率
3. 积累结构化中医案例数据，支持科研和传承
4. 降低患者获取优质中医服务的门槛

## 2. 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|---------|
| **医生** | 中医师/中西医结合医师 | 四诊采集、AI辅助诊断、开方、查看历史病历 |
| **患者** | 就诊用户 | 查看诊断报告、治疗方案、健康建议 |
| **系统管理员** | IT运维人员 | 系统配置、设备管理、模型管理、数据统计 |
| **AI训练师** | 中医专家/数据科学家 | 标注数据、评估模型输出、调优知识库 |

### 3.2 患者管理模块

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **患者建档** | 姓名、性别、年龄、联系方式、身份证号 | P0 |
| **中医体质档案** | 九种体质辨识结果存储 | P1 |
| **病史管理** | 既往病史、过敏史、家族病史、手术史 | P0 |
| **就诊记录** | 按时间线展示所有就诊记录，支持检索 | P0 |
| **检查检验** | 西医检查化验单上传与归档 | P2 |

### 3.3 四诊采集模块

#### 3.3.1 望诊

| 功能 | 数据来源 | 采集内容 | 优先级 |
|------|---------|---------|--------|
| 舌诊采集 | 高清舌诊仪/手机摄像头 | 舌质、舌苔、舌形、舌下脉络高清影像 | P0 |
| 面色采集 | 高清摄像头 | 面色、神采、面部特征 | P0 |
| 形态观察 | 摄像头/人工 | 体型、步态、姿态 | P1 |
| 目诊/耳诊 | 专科设备 | 巩膜、耳廓等微诊信息 | P2 |

#### 3.3.2 闻诊

| 功能 | 数据来源 | 采集内容 | 优先级 |
|------|---------|---------|--------|
| 声诊采集 | 高保真麦克风 | 语音频率、音色、气息特征 | P0 |
| 气味采集 | 电子鼻/人工 | 口气、体味特征（人工标注） | P2 |
| 咳声分析 | 麦克风 | 咳嗽频率、深浅、痰音特征 | P1 |

#### 3.3.3 问诊

| 功能 | 数据来源 | 采集内容 | 优先级 |
|------|---------|---------|--------|
| 智能问诊表单 | 系统界面 | 十问歌标准化问卷（寒热、汗、头身、二便等） | P0 |
| 症状自述录入 | 医生录入/语音转文字 | 患者主诉、现病史 | P0 |
| 智能问答引导 | AI引擎 | 根据已填信息动态追问 | P1 |

#### 3.3.4 切诊

| 功能 | 数据来源 | 采集内容 | 优先级 |
|------|---------|---------|--------|
| 脉诊采集 | 脉诊仪（如喜黄、智脉等） | 脉位、脉率、脉形、脉势、节律 | P0 |
| 脉象图谱 | 脉诊仪 | 脉波图、时域/频域参数 | P0 |
| 腹诊/穴诊 | 触诊设备/人工 | 腹部压痛点、穴位敏感度 | P2 |

| 功能 | 描述 | 优先级 |

| **外治法建议** | 拔罐、刮痧、推拿等建议 | P2 |

| **疗效预测** | 基于相似病例库预测治疗效果 | P3 |

## 4. 非功能需求

### 4.1 性能需求
- 四诊数据采集延迟 < 200ms
- AI诊断响应时间 < 10s（允许异步处理复杂分析）
- 支持并发诊疗 ≥ 50个诊室同时使用
- 历史病历检索 < 2s

### 4.2 安全需求
- 患者医疗数据全程加密存储（AES-256）
- 网络传输采用 TLS 1.3
- 符合《信息安全技术 健康医疗数据安全指南》(GB/T 39725)
- 系统操作全程审计日志
- 用户认证支持双因素认证

### 4.3 可用性需求
- 系统可用性 ≥ 99.5%
- 支持离线模式（网络中断时本地存储，恢复后同步）
- 数据自动备份频率：每15分钟增量备份，每日全量备份

### 4.4 兼容性需求
- 支持主流四诊仪厂商设备接入（开放式SDK标准）
- 桌面端支持 Windows 10/11，推荐触控屏
- 支持主流浏览器（Chrome/Edge）

### 4.5 合规需求
- 符合《中医药法》及《中医诊所备案管理暂行办法》
- 软件定位为"辅助诊断"，不替代医师独立判断
- 患者数据跨境传输限制（数据本地化存储）

## 5. 用户故事

### 医生视角
```
作为一位中医师，我希望连接脉诊仪和舌诊仪采集患者客观数据，
结合我的问诊信息，一键生成AI辅助诊断报告，
这样我可以更快、更准确地完成辨证开方。
```

### 患者视角
```
作为一位患者，我希望通过手机APP填写预问诊信息，
到诊室后医生直接查看我的信息进行诊断，
并能在手机上查看我的诊断报告和中药方剂。
```

### 管理者视角
```
作为医院管理者，我希望查看全院的中医诊疗数据统计，
了解证型分布、用药排名、治疗效果趋势，
为科室管理和科研提供数据支撑。
```

## 6. 业务流程

### 6.1 标准诊疗流程
```
患者挂号 → 预问诊（APP/小程序） → 到诊 → 
望诊采集（舌诊仪+面诊） → 闻诊采集（声诊） → 
切诊采集（脉诊仪） → 医生问诊（系统辅助） → 
AI综合诊断（证型+方剂） → 医生审核/调整 → 
生成治疗方案 → 打印/电子交付 → 
复诊提醒
```

### 6.2 数据同步流程
```
采集端（诊所） → 本地缓存 → 加密传输 → 
云端处理（AI模型推理） → 结果返回 → 
本地展示 → 归档存储
```

## 7. 验收标准

| 编号 | 验收项 | 标准 |
|------|-------|------|
| AC-1 | 脉诊仪连接 | 支持主流脉诊仪，数据读取延迟 < 500ms |
| AC-2 | 舌诊采集 | 图片清晰度 > 1080P，自动舌体分割准确率 > 90% |
| AC-3 | AI证型诊断 | Top-3 证型命中率 > 85%（对照专家标注） |
| AC-4 | 方剂推荐 | 推荐方剂与专家处方吻合率 > 80% |
| AC-5 | 系统并发 | 50个诊室同时操作，平均响应 < 3s |
| AC-6 | 离线模式 | 断网4小时内数据不丢失，恢复后自动同步 |
